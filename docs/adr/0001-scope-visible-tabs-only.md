# 0001. 作用域：全局开关、仅对可见标签页生效、不可见即暂停

- 状态（Status）：已接受（Accepted）
- 日期（Date）：2026-06-29

## 背景（Context）

本扩展的目标是**活跃伪装**（见 `CONTEXT.md`）：在**观察渠道 = 屏幕画面**（屏幕共享 / 投屏 / 肉眼偷瞄，即场景 A/D）下，让屏幕画面持续呈现"有人在浏览网页"的样子。

由此推出一个根本约束：**只有显示在屏幕上的画面才会被旁观者看到。** 标签页若未显示（切到后台、最小化、锁屏），无论是否滚动，旁观者都看不见，伪装也就没有意义。

在"作用域"上存在一个真实的设计岔路：

- **A 方案 · 全局 + 仅可见标签页**：一个全局开关；任何当前可见的标签页都自动滚动；标签页不可见即暂停。
- **B 方案 · 单一指定标签页**：用户指定某一个"摸鱼标签页"，仅该标签页滚动，与可见性解耦。

## 决策（Decision）

采用 **A 方案**：

1. 开关是**全局**的（存于 `chrome.storage.local`，对所有标签页一致）。
2. 自动滚动**只在可见标签页**（`document.visibilityState === "visible"`）运行。
3. 标签页变为不可见时，**暂停滚动并释放 Screen Wake Lock**；重新可见时自动恢复并重新申请。

## 权衡与理由（Rationale / Trade-offs）

- 在观察渠道 = 屏幕画面的前提下，"对可见标签页生效"与"对被观察的画面生效"天然等价，因此无需引入"指定目标标签页"的概念。
- A 方案**无需跨标签通信、无需标签页注册表**，实现最简单、最易维护。
- B 方案能精确控制"只滚某一个页面"，但需要标签页追踪 + 消息架构，复杂度显著更高，而它带来的精确性在本场景下收益很低。

## 后果（Consequences）

- ✅ 实现简单，无后台空转（不可见即停，省资源）。
- ⚠️ **使用约定**：要摸鱼的标签页必须保持**前台可见**；切走 / 最小化 / 锁屏会让画面静止，反而暴露。
- ⚠️ 开启期间，**任何**可见标签页都会滚动；真正干活时需手动关闭开关。
- 🔁 若未来确有"只滚指定标签页"的需求，需要新开一条 ADR 推翻本决策，并引入标签页注册与跨标签通信。
