# 摸鱼助手 · 自动滚动防屏保浏览器插件

一个 Chrome / Edge（Manifest V3）浏览器扩展。开启后会**自动、平滑地滚动当前页面**，并在滚动过程中**随机停顿**（模拟真人阅读），同时**模拟鼠标移动**、申请 **Screen Wake Lock 保持屏幕常亮**。

适合：暂时离开但又不想让屏幕进入屏保 / 锁屏、或让页面看起来一直“有人在操作”的场景。

> ⚠️ 请遵守你所在公司或平台的规定，本工具仅供学习与个人使用。

---

## ✨ 功能

- **自动滚动**：平滑向下滚动，到底部后自动“来回滚动”或“回到顶部循环”。
- **拟人停顿**：可设置停顿概率与时长区间，模拟真人阅读时的停顿。
- **拟人化抖动**：每一步滚动距离带有随机微调，避免机械式匀速。
- **模拟鼠标移动**：在页面内派发 `mousemove` 事件，让站点判定为“活跃”。
- **保持屏幕常亮**：使用浏览器原生 [Screen Wake Lock API](https://developer.mozilla.org/docs/Web/API/Screen_Wake_Lock_API) 阻止显示器息屏 / 屏保。
- **实时生效**：所有设置在弹窗中调整后立即生效，无需重新开关。
- **后台友好**：标签页不可见时自动暂停（节省资源），切回时自动恢复并重新申请常亮锁。

## 📦 安装（开发者模式加载）

本扩展未上架商店，需手动加载：

1. 下载/克隆本仓库。
2. 打开 Chrome / Edge，地址栏输入：
   - Chrome: `chrome://extensions`
   - Edge: `edge://extensions`
3. 打开右上角 **开发者模式 (Developer mode)**。
4. 点击 **加载已解压的扩展程序 (Load unpacked)**。
5. 选择本仓库中的 **`extension/`** 文件夹。
6. 在工具栏固定 ☕️ 图标，点击即可打开控制面板。

---

## 🕹 使用

点击工具栏的 ☕️ 图标打开控制面板：

| 设置 | 说明 |
| --- | --- |
| **总开关** | 启动 / 停止摸鱼。开启后徽标显示 `ON`。 |
| **滚动速度** | 每一步移动的像素数，越大越快。 |
| **平滑度** | 滚动步进的时间间隔（ms），越小越顺滑。 |
| **到底部时** | `来回滚动` 或 `回到顶部循环`。 |
| **停顿概率** | 一段滚动后进入停顿的概率。 |
| **停顿最短 / 最长** | 停顿时长的随机区间。 |
| **模拟鼠标移动** | 在页面内派发 `mousemove` 事件。 |
| **保持屏幕常亮** | 申请 Wake Lock，阻止息屏 / 屏保。 |
| **拟人化抖动** | 给每步滚动加入随机微调。 |
| **恢复默认** | 一键还原默认参数（不影响开关状态）。 |

设置会保存在 `chrome.storage.local`，并对所有标签页生效。

## 🧩 工作原理

```
extension/
├── manifest.json      # MV3 清单
├── defaults.js        # 共享默认设置（content / popup / background 复用）
├── background.js      # Service Worker：初始化设置、维护徽标
├── content.js         # 注入每个页面的“引擎”：滚动 / 停顿 / 鼠标 / Wake Lock
├── popup.html/css/js  # 控制面板 UI
├── _locales/          # 中 / 英 文案
└── icons/             # 图标（由 generate_icons.py 生成）
```

- 设置存于 `chrome.storage.local` 的 `settings` 键；`content.js` 监听 `storage.onChanged`，调整后**实时重启引擎**。
- `content.js` 用基于 `setTimeout` 的循环驱动滚动，便于在“滚动 / 停顿”之间灵活切换时序。

### 重新生成图标

```bash
cd extension/icons
python3 generate_icons.py
```

## ⚠️ 重要说明与限制

浏览器扩展运行在**沙箱**中，有几点物理限制需要明确：

1. **无法移动操作系统真实鼠标光标**，也无法直接重置操作系统的“空闲计时器”。扩展只能在网页内部派发合成的 `mousemove` 事件，这对**网页 / 网站**的“活跃判定”有效，但**不会**让系统以为你在用鼠标。
2. **阻止屏保 / 息屏靠的是 Wake Lock API**，而不是鼠标移动。这是浏览器提供的、唯一可靠地让**屏幕保持常亮**的网页接口。它需要：
   - **HTTPS 等安全上下文**（普通 `http://`、部分 `file://` 页面可能不可用）；
   - **标签页处于可见状态**（标签页被切到后台时锁会自动释放，切回时本扩展会自动重新申请）。
3. 若你需要的是**让操作系统层面认为“有人在操作电脑”**（例如阻止 Windows / macOS 自身的锁屏策略），单靠浏览器扩展通常无法做到，需要桌面端工具（不在本扩展范围内）。

简言之：**本扩展能让网页持续滚动 / 看起来活跃，并通过 Wake Lock 让屏幕不息屏**；但它不能伪造系统级鼠标输入。

## 🔒 隐私

- 不收集、不上传任何数据。
- 仅申请 `storage` 权限用于保存设置；通过清单中静态声明的内容脚本（`<all_urls>`）在页面内实现自动滚动。
