# 摸鱼助手 · 自动滚动假装在看

一个 Chrome / Edge（Manifest V3）浏览器扩展。它的**唯一目标**是：当别人通过**屏幕画面**观察你（屏幕共享 / 投屏 / 旁人偷瞄）时，让画面持续呈现出"**有人正在浏览网页**"的样子——也就是**活跃伪装**。

开启后，它会自动、平滑地滚动当前页面，并随机停顿（模拟真人阅读）；可选申请 **Screen Wake Lock** 作为辅助，避免息屏 / 屏保把滚动画面盖掉。

> ⚠️ 仅供学习与个人使用，请遵守你所在公司或平台的规定。

---

## 它能做什么、不能做什么（先看这里）

本扩展只针对一种**观察渠道**：别人看的是你**屏幕上显示的画面**。

| 场景 | 别人怎么看你 | 本扩展 |
| --- | --- | --- |
| **A 屏幕共享 / 投屏** | 用眼睛看你的屏幕画面 | ✅ 适用：画面持续滚动，像有人在看 |
| **D 旁人肉眼偷瞄** | 路过扫一眼你的屏幕 | ✅ 适用：同上 |
| **B 操作系统级监控** | OS API 统计鼠标键盘空闲 / 定时截图 | ❌ 无效：需桌面端工具 |
| **C 网页在线状态** | 网页用自身心跳/事件判在线 | ❌ 不以此为目标 |

**两条硬边界，务必知晓：**

1. **不会移动真实鼠标光标。** 浏览器扩展无法移动操作系统的鼠标指针，所以旁观者看到的**真实光标始终静止**。本扩展靠的是**画面在滚动**，而不是"鼠标在动"。
2. **可信度上限 = 扫一眼可信。** 它能骗过"瞥一眼"的人；但若有人**长时间紧盯**，会看出是在**反复滚动同一页面**。本扩展刻意**不做**自动点击 / 自动换页等高风险拟人手段。

---

## 使用约定（满足这些它才有效）

- **摸鱼的标签页必须保持前台可见。** 切到别的窗口、最小化、锁屏走开，都会让滚动暂停、画面静止——反而暴露。
- **开关是全局的：开启后任何可见标签页都会滚动。** 真正干活时请先关掉开关。详见 [ADR 0001](./docs/adr/0001-scope-visible-tabs-only.md)。
- **建议开在长内容页面**（信息流、长文档、长列表）。短页面来回弹最容易看假。

## 📦 安装（开发者模式加载）

1. 下载 / 克隆本仓库。
2. 打开 `chrome://extensions`（Edge 为 `edge://extensions`）。
3. 打开右上角 **开发者模式 (Developer mode)**。
4. 点击 **加载已解压的扩展程序 (Load unpacked)**，选择本仓库的 **`extension/`** 文件夹。
5. 把 ☕️ 图标固定到工具栏，点击打开控制面板。

## 🕹 使用

点击工具栏 ☕️ 图标打开控制面板：

| 设置 | 说明 |
| --- | --- |
| **总开关** | 启动 / 停止。开启后徽标显示 `ON`。 |
| **滚动速度** | 每一步移动的像素数，越大越快。 |
| **平滑度** | 滚动步进的时间间隔（ms），越小越顺滑。 |
| **到底部时** | `来回滚动`（更自然，推荐）或 `回到顶部循环`。 |
| **停顿概率** | 一段滚动后进入停顿的概率。 |
| **停顿最短 / 最长** | 停顿时长的随机区间。 |
| **真人接管** | 检测到真人操作时如何让位（见下）。 |
| **接管阈值** | "持续操作超过 N 秒"模式下，连续操作多久才判定接管。 |
| **离开后自动恢复** | 真人离开（无操作）一段时间后自动恢复滚动。 |
| **恢复等待** | 自动恢复前需要的"无操作"时长。 |
| **拟人化抖动** | 给每步滚动加入随机微调，避免机械匀速。 |
| **保持屏幕常亮（辅助）** | 申请 Wake Lock，避免息屏 / 屏保盖住滚动画面。 |
| **调试日志（控制台）** | 在网页控制台打印诊断日志，便于排查不生效的页面。 |
| **恢复默认** | 还原默认参数（不影响开关状态）。 |

设置保存在 `chrome.storage.local`，调整后实时生效。

### 真人接管（你回来了，工具自动让位）

为了避免"你回来操作时它还在自顾自滚动"，扩展会检测**真实**的鼠标移动 / 滚轮 / 按键 / 触摸，三种模式：

- **关闭**：不检测，一直滚。
- **动一下就暂停**（默认）：一旦有真人操作，立即暂停。
- **持续操作超过 N 秒才暂停**：手抖、误碰（短于 N 秒）不会触发，连续操作超过阈值才判定为真人接手。

接管后：

- **离开后自动恢复 = 开**：暂停滚动；当你重新离开、且"无操作"达到「恢复等待」时长后，自动恢复滚动。适合"回来处理一下又走开"的循环。
- **离开后自动恢复 = 关**：直接**关闭总开关**（硬停止），需要你手动重新开启。

> 说明：页面自动滚动时，静止鼠标下方的元素会变化、从而触发 `mousemove`。扩展会通过判断指针坐标是否真的改变来**排除这类伪移动**，不会把自己的滚动误判成真人接管。

### 短页面行为

如果当前页面（或滚动容器）的可滚动高度太短（默认小于 `160px`），来回快速弹动会非常假。此时扩展会**自动暂停滚动并定期重新检测**（页面可能懒加载出更多内容），而不是机械地上下弹。建议把扩展开在**长内容页面**。

> 提示：扩展会自动识别真正的滚动容器——既支持整页滚动，也支持很多单页应用（SPA）里"内部某个滚动区域"的情况。

## 🧩 工作原理

```
extension/
├── manifest.json      # MV3 清单（仅申请 storage 权限）
├── defaults.js        # 共享默认设置
├── background.js      # Service Worker：初始化设置 + 维护徽标
├── content.js         # 注入页面的引擎：滚动 / 停顿 / Wake Lock
├── popup.html/css/js  # 控制面板 UI
├── _locales/          # 中 / 英 文案
└── icons/             # 图标（由 generate_icons.py 生成）
```

- 设置存于 `chrome.storage.local` 的 `settings` 键；`content.js` 监听 `storage.onChanged`，调整后**实时重启引擎**。
- `content.js` 用基于 `setTimeout` 的循环驱动滚动，便于在"滚动 / 停顿"之间灵活切换时序。
- 标签页不可见时**自动暂停并释放 Wake Lock**，重新可见时恢复（见 [ADR 0001](./docs/adr/0001-scope-visible-tabs-only.md)）。

### 重新生成图标

```bash
cd extension/icons
python3 generate_icons.py
```

## 🩺 某些页面不生效？这样排查

弹窗底部的「**当前页面诊断**」面板会实时显示原因，点「刷新」可重新检测：

- **"无法在此页面运行"**：这是**受限页面**，内容脚本无法注入。常见于：
  - 浏览器内置页面：`chrome://`、`edge://`、设置页、扩展管理页、扩展商店；
  - PDF 阅读器、`view-source:`；
  - 本地 `file://` 页面（除非在扩展详情里手动允许"访问文件网址"）。
  - → 换一个**普通网页**即可。
- **"可滚动区域不足"**：页面内容太短（见上文"短页面行为"），换长页面。
- **"屏幕常亮未生效 / 非 HTTPS"**：Wake Lock 要求 **HTTPS 安全页面**且标签页可见；`http://` 页面会失败（不影响滚动本身）。
- **想看更详细的原因**：在弹窗里打开「**调试日志（控制台）**」，然后在该网页按 `F12` 打开开发者工具的 **Console**，筛选 `[摸鱼助手]` 前缀，即可看到滚动容器识别、Wake Lock 申请、接管/恢复等详细日志。

> 内容脚本以 `<all_urls>` 注入，但浏览器**禁止**在上述受限页面注入任何扩展脚本——这是浏览器的安全限制，并非本扩展的缺陷。
